# 🤖 Claude Auto Developer - 遠端任務管理中心

> **這是 Claude Auto Developer 的遠端任務同步檔案**  
> 編輯此檔案來管理你的開發任務，系統會自動同步到本地並執行開發  
> GitHub 儲存庫: https://github.com/austinchang/claude-auto-developer-tasks  

---

## 📋 使用說明

### 🔄 自動同步機制
- **本地監控**: 系統每 5 分鐘檢查此檔案的變化
- **差異檢測**: 發現變化時自動下載並比較差異
- **自動執行**: 新增的任務會自動排程執行
- **狀態更新**: 完成的任務會自動更新狀態並同步回 GitHub

### 📝 任務格式
```markdown
- [ ] **任務標題** | 截止日期: XXX | 技術: XXX | 描述: XXX | 時間: XXX
```

### 🏷️ 狀態標記
- `- [ ]` 待執行 (新任務)
- `- [~]` 執行中 (系統自動標記)  
- `- [x]` 已完成 (系統自動標記)
- `- [!]` 失敗 (系統自動標記)

### 📊 優先級分類
- 🔥 **緊急** (24小時內) - 系統會優先處理
- ⭐ **高優先級** (本週內) - 正常排程執行  
- 📅 **一般** (兩週內) - 背景執行
- 🔖 **低優先級** (一個月內) - 閒置時執行

## 🎯 適合自動化的任務類型

### ✅ **強烈推薦**
- **Web 應用開發**: React、Vue、Angular 前端專案
- **API 後端服務**: FastAPI、Express、Django REST 
- **數據處理工具**: Pandas、NumPy 數據分析
- **自動化腳本**: 檔案處理、系統管理、爬蟲
- **小型工具**: CLI 工具、GUI 應用、實用程式

### ⚠️ **需要注意**
- **複雜業務邏輯**: 需要明確詳細的需求描述
- **第三方整合**: 需要提供 API 金鑰和文檔連結
- **資料庫設計**: 需要明確的資料結構和關聯

### ❌ **不適合**
- **需要人工決策**: 設計評審、需求確認、業務決策
- **需要測試確認**: 生產環境部署、用戶驗收測試
- **需要外部協調**: 需要與他人協作或確認的任務

```markdown
- [ ] **做一個網站** | 截止日期: 下週 | 技術: 看情況 | 描述: 需要一個網站 | 時間: 不確定
```

**為什麼不好:**
- ❌ 範圍過於模糊
- ❌ 技術選擇不明確
- ❌ 缺乏具體需求
- ❌ 時間估計不確定

## 🚀 快速開始

### 1. **編輯此檔案**
在 GitHub 上直接編輯，或者 clone 到本地編輯後推送

### 2. **添加你的任務**
使用上述格式添加新任務到對應的優先級分類中

### 3. **等待自動同步**
系統會在 5 分鐘內檢測到變化並開始執行

### 4. **監控進度**
檢查本地系統狀態或等待 GitHub 上的狀態更新

### 5. **查看結果**
完成的項目會自動推送到對應的 GitHub 儲存庫
